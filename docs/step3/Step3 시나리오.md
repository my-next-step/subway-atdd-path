# Step3 시나리오 

**정상적인 시나리오**
```properties
Feature: 지하철 경로 찾기 기능

    Scenario: 출발역과 도착역간의 최단 경로와 거리 조회
      Given 지하철 노선 생성을 요청 하고
      And 지하철 노선에 새로운 구간을 등록하고
      When 출발역과 도착역간의 최단 경로 찾기 요청을 하면 
      Then 출발역과 도착역간의 최단 경로와 거리를 조회할 수 있다.
```

**비정상적인 시나리오**  
```properties
Feature: 지하철 경로 찾기 기능

    Scenario: 출발역과 도착역이 같은 경우의 최단 경로와 거리 조회
      Given 지하철 노선 생성을 요청 하고
      And 지하철 노선에 새로운 구간을 등록하고
      When 출발역과 도착역이 같은 경우의 최단 경로 찾기 요청을 하면 
      Then 출발역과 도착역간의 최단 경로와 거리를 조회할 수 없다.

    Scenario: 출발역과 도착역이 연결이 되어 있지 않은 경우의 최단 경로와 거리 조회
      Given 지하철 노선 생성을 요청 하고
      And 지하철 노선에 새로운 구간을 등록하고
      When 출발역과 도착역이 연결이 되어 있지 않은 경우의 최단 경로 찾기 요청을 하면 
      Then 출발역과 도착역간의 최단 경로와 거리를 조회할 수 없다.

    Scenario: 존재하지 않은 출발역이나 도착역을 조회 할 경우의 최단 경로와 거리 조회
      Given 지하철 노선 생성을 요청 하고
      And 지하철 노선에 새로운 구간을 등록하고
      When 존재하지 않은 출발역이나 도착역을 조회 할 경우의 최단 경로 찾기 요청을 하면 
      Then 출발역과 도착역간의 최단 경로와 거리를 조회할 수 없다.
```
